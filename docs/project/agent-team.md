# 에이전트 역할 기준

## 팀장

- 목표와 우선순위를 정리합니다.
- 권한, 개인정보, 데이터 안정성, 배포 반영 여부를 최종 확인합니다.
- 다른 에이전트 결과물을 승인하거나 재작업 지시합니다.
- Notion 인수인계와 repo 문서를 기준으로 작업 흐름을 유지합니다.

## 프로그래밍

- Version.3 Next UI, 별도 서버 API, 권한/세션/감사 로그 계약을 구현합니다.
- 타입, 린트, Pages 빌드, 운영 표면 검증, 서버 계약 검증을 통과시킵니다.
- 저장 실패, 권한 차단, 세션 만료를 명확하게 처리합니다.
- Base44 또는 Lovable 서버 후보를 붙일 때도 `verify:version3-server`를 통과 기준으로 삼습니다.

## 디자인

- `public/brand/bonsung-design-template.png`를 기준 자산으로 사용합니다.
- 모바일은 데스크톱 축소가 아니라 역할별 실제 업무 순서에 맞춘 앱형 GUI로 설계합니다.
- 목록, 상세, 예약, 설정 화면의 일관된 흐름을 우선합니다.
- build-web-apps는 실제 화면 구현과 반응형 검증에, creative-production은 브랜드 자산과 시각 방향 탐색에 사용합니다.

## 데이터관리

- Version.3 서버 엔티티, 상태값, 필수 입력 기준을 정리합니다.
- 필수값, 중복, 참조 오류, 예약 충돌, 감사 로그 검증 기준을 관리합니다.
- 운영 백업과 복구 절차를 문서화합니다.
- Base44를 쓰는 경우 엔티티/권한/백엔드 함수 설계가 Version.3 서버 계약과 맞는지 먼저 확인합니다.

## 업무기록

- Notion의 작업 현황, 인수인계, 검증 결과를 repo 문서와 맞춰 기록합니다.
- 완료 보고에는 변경 요약, 변경 파일, 검증 결과, 알려진 제한사항, 다음 작업 제안을 포함합니다.
- template-creator는 반복 보고서, 운영 점검표, 회의록처럼 형식이 반복되는 산출물 템플릿을 만들 때 사용합니다.

## 플러그인 활용 기준

- `base44`: 실제 운영 서버 후보를 만들거나 데이터 모델, 백엔드 함수, 권한 규칙을 구현할 때 우선 검토합니다.
- `lovable`: 빠른 풀스택 후보 화면 또는 서버 후보를 만들어 Version.3 계약과 비교할 때 사용합니다.
- `build-web-apps`: Next UI, 반응형 운영 화면, 브라우저 검증 중심 작업에 사용합니다.
- `creative-production`: 본성뮤직 브랜드 이미지, 안내 자료, 시각 방향, 홍보성 화면 자산을 탐색할 때 사용합니다.
- `template-creator`: Notion/문서/점검표처럼 반복 사용 가능한 운영 양식을 만들 때 사용합니다.
