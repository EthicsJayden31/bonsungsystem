# 운영 기능 v3

## 추가 기능

- 등록·재등록 결제 이력과 다음 결제 예정일
- 관리자·직원용 14일 이내 재등록 예정 목록
- 레슨실 4개와 연습실 3개의 역할별 예약
- 공간 이름 변경과 전체 예약 상태 관리
- 관리/직원/수강생 대분류와 원장/팀장/사원/강사 직급
- 관리자와 원장의 계정·개별 권한 관리
- 직원·강사의 출퇴근 기록
- 원장·팀장의 회의 예약과 참석자 일정
- 수업·회의·휴무·행사를 합친 통합 캘린더
- 기기 설정/라이트/다크 화면 모드
- 로그인 인증과 화면 데이터 로딩을 분리한 단계적 bootstrap 요청

## 권한 기준

| 구분 | 기본 권한 |
| --- | --- |
| 관리자 | 전체 기능과 이용 현황, 계정·권한 관리 |
| 원장(대표) | 전체 운영, 계정·권한, 회의, 캘린더 관리 |
| 팀장 | 운영 데이터, 회의, 캘린더 관리 |
| 사원 | 수강생·등록·공간 등 일상 운영 관리 |
| 강사 | 담당 수업·수강생·수업일지, 레슨실 예약, 본인 근태와 회의 |
| 수강생 | 본인 수강·일지·등록 이력, 연습실 예약, 개인 캘린더 |

관리자와 원장은 계정별로 운영 관리, 회의, 캘린더, 수납 조회 권한을 조정할 수 있다.

## Google Sheets 시트

기존 시트는 유지하고 다음 시트를 자동 생성한다.

- `등록결제`
- `공간`
- `공간예약`
- `근태`
- `회의`
- `학원일정`

`공간` 시트가 비어 있으면 대형 레슨실 1, 중형 레슨실 1, 소형 레슨실 1·2, 연습실 A·B·C가 자동 생성된다.

## 로컬 기능 테스트

정적 파일 서버에서 다음 주소를 연다.

```text
http://localhost:포트/test.html
```

테스트 화면은 운영 API를 호출하지 않고 브라우저 저장소만 사용한다. 상단 계정 전환 메뉴에서 관리자, 원장, 팀장, 사원, 강사, 수강생 화면을 즉시 전환할 수 있다.

샘플 수강생, 수강 등록, 수업, 수업일지, 결제, 공간 예약, 근태, 회의, 휴무와 행사 데이터가 포함되어 있다. `초기화` 버튼으로 언제든 기본 테스트 데이터로 되돌릴 수 있다.

운영 주소인 `index.html`에는 테스트 계정 전환 도구가 표시되지 않는다.

## 성능 변경

- 로그인 성공 시 인증과 계정 정보만 먼저 반환해 화면 전환을 지연시키지 않는다.
- 첫 화면은 `bootstrap` 한 번으로 대시보드·캘린더·재등록 핵심 데이터만 불러온다.
- 예약, 근태, 회의, 수업일지, 이용 현황은 해당 메뉴를 열 때 필요한 데이터만 추가로 불러온다.
- 같은 요청 안에서 읽은 시트 데이터는 메모리에 재사용한다.
- 스키마 확인 결과는 6시간 캐시해 모든 API 요청에서 시트 구조를 반복 검사하지 않는다.
