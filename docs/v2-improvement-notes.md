# 개선 버전 설계 메모

## 이번 버전의 방향

- 운영자가 빠르게 훑을 수 있는 조용하고 밀도 있는 업무 화면
- Google Sheet 연결 절차를 일반 사용자에게 노출하지 않는 자동 연결
- 역할에 따라 필요한 정보만 보여 주는 개인 대시보드
- 수업 직후 짧은 시간 안에 작성할 수 있는 수업일지 흐름
- 모바일에서 좌우 이동 없이 한 방향으로 읽고 입력하는 레이아웃

## 적용한 UX 원칙

### 모바일 리플로

320 CSS px 너비에서 두 방향 스크롤이 필요하지 않도록 표를 행 단위 카드로 바꾼다. 각 셀은 원래 열 제목을 라벨로 표시하고 긴 문자열은 줄바꿈한다.

### 입력 목적 표시

로그인, 이메일, 전화번호, 비밀번호 입력에 브라우저가 이해할 수 있는 `autocomplete` 값을 사용한다. 반복 입력과 모바일 키보드 전환 비용을 줄인다.

### 수업일지 작성 속도

- 이전 일지의 목표·과제를 불러오기
- 자주 쓰는 템플릿 적용
- 입력 중 브라우저 자동 임시 저장
- 수강생 선택 시 과목과 담당 강사 자동 채우기
- 출결 상태를 한 번에 선택하는 분할 버튼

### 운영 분석

이용 기록은 로그인, 페이지 조회, 계정·수강생·수강·수업·수업일지 변경을 이벤트로 남긴다. 관리자는 최근 14일 활동량, 최근 7일 사용자, 활동 유형과 최신 기록을 확인한다.

## 기술 근거

- Apps Script 웹앱은 `doGet`·`doPost`를 제공하고 소유자 권한으로 실행되도록 배포할 수 있어 정적 GitHub Pages와 비공개 Sheet 사이의 API 계층으로 사용한다.
- WCAG 2.2 Reflow 기준을 참고해 320 CSS px에서 정보 손실이나 양방향 스크롤이 생기지 않도록 한다.
- WCAG Identify Input Purpose 기준을 참고해 개인 정보 입력 필드의 목적을 브라우저가 식별하도록 한다.
