# 모바일·연결형 운영 기능 v4

## 모바일 사용성

- 화면 하단 양쪽에 반투명 `이전 화면`, `맨 위로` 버튼을 표시한다.
- 수업일지는 `일지 조회`와 `일지 작성`으로 분리한다.
- 수강생, 수강·일정, 공간 예약, 계정 관리도 작업별 하위 메뉴를 제공한다.
- 모바일 입력 요소는 16px 이상을 사용해 포커스 시 화면이 자동 확대되지 않게 한다.
- 390px와 320px에서 문서 너비와 스크롤 너비가 같도록 세로형 레이아웃을 사용한다.

## 공간 예약 규칙

| 계정 | 조회·예약 공간 | 예약 목적 | 취소 범위 |
| --- | --- | --- | --- |
| 관리자·원장·팀장·사원 | 전체 공간 | 레슨, 이론수업, 회의, 연습 | 전체 |
| 강사 | 레슨실, 연습실 | 레슨, 이론수업, 연습 | 본인 예약 |
| 수강생 | 연습실 | 연습 | 본인 예약 |

- 예약은 정각부터 정확히 1시간 단위다.
- 공간 구역도와 시간별 점유 카드에서 예약 상태를 확인한다.
- 공간명을 누르면 해당 공간의 예정 사용 일정을 확인한다.

## 연결형 상세 화면

- 관리자, 원장, 팀장은 직원·강사·수강생 이름을 눌러 프로필을 연다.
- 직원 프로필은 참석 회의와 본인 업무를 표시한다.
- 강사 프로필은 예정 수업, 담당 수강생, 작성 수업일지를 표시한다.
- 수강생 프로필은 예정 수업, 수강 이력, 수업일지를 표시한다.
- 캘린더 수업과 회의를 누르면 해당 상세 화면으로 이동한다.
- 수업 상세에서 회차, 결석·취소 사유, 보강일, 내부 메모를 관리한다.

## 계정과 권한

- `admin`은 시스템 관리 계정이며 회의 참석자와 직원 명단에서 제외한다.
- 관리자와 원장은 계정 구분, 직원 직급, 연결 수강생, 연락처를 변경할 수 있다.
- 계정 목록과 편집 화면에서 로그인 횟수, 최근 로그인과 상세 로그인 이력을 확인한다.
- 계정 구분을 변경하면 역할별 기본 권한을 다시 적용하고 기존 세션을 종료한다.
- 개별 권한은 수강생 관리, 수업일지 작성, 전체 예약 관리, 회의, 캘린더, 수납 조회 단위로 조정한다.

## 새 Google Sheets 데이터

- 기존 `수업일정`에 `absence_reason`, `makeup_date`, `lesson_number` 열을 추가한다.
- 기존 `수업일지`에 `absence_reason`, `makeup_date`, `lesson_number` 열을 추가한다.
- `업무` 시트를 추가한다.
- `수업종류` 시트를 추가한다.
- 스키마 버전은 `4`다.

Apps Script 코드를 배포한 뒤 웹앱 배포의 새 버전을 만들어야 운영 페이지에서 새 API가 사용된다.

## 로컬 검증

```text
http://localhost:4173/test.html
```

상단 계정 전환에서 관리자, 원장, 팀장, 사원, 강사, 수강생을 선택한다. `초기화`를 누르면 v4 샘플 수업 회차, 결석·보강, 타인 예약, 업무, 수업 종류 데이터가 복원된다.
