본성뮤직 Version.3 구축 현황 점검

공개 UI, 별도 Version.3 서버, 실제 운영 데이터 연결 상태를 한 화면에서 확인하기 위한 임시 페이지입니다. 최종 운영 매뉴얼이 아니라, 다음 배포 단계로 넘어가기 전 빠르게 확인하는 점검판입니다.

1. Version.3 서버 연결 확인

서버 주소가 정해지면 ?api=https://서버주소를 붙여 바로 점검할 수 있습니다.

아직 서버 상태를 확인하지 않았습니다.

공개 UI

온라인 확인 완료 GitHub Pages의 로그인, 데이터 점검, 임시 점검 페이지가 200 응답을 반환합니다.

서버 기준

별도 API 필요 Apps Script가 아니라 Version.3 서버가 로그인과 운영 데이터를 담당해야 합니다.

점검 상태

대기 서버 health 응답을 확인하면 상태가 바뀝니다.

2. 화면 바로 열기

로그인 대시보드 계정 관리 학생 관리 상담요청 데이터 점검 오프라인 점검판

3. 운영 흐름 기준

  • 상담·등록에서 수강생, 수업 기록, 프로그램/강사 운영으로 이어져야 합니다.
  • 등록 확인, 수납 체크, 문서 정리, 권한 관리, 매뉴얼 업데이트가 한 흐름으로 보여야 합니다.
  • 대표, 매니저, 강사, 수강생 역할별로 볼 수 있는 정보와 저장 권한이 달라야 합니다.
  • 상담요청은 Q&A가 아니라 한 방향 요청과 상태 관리로 처리합니다.
  • 공지는 대표와 매니저 권한에서만 작성할 수 있어야 합니다.

4. 외부 서버 배포 후보

  1. 현재 최단 경로는 GHCR 서버 이미지를 Render 같은 Docker 호스팅에 올리는 방식입니다.
  2. Base44에는 현재 본성/Bonsung 기존 앱이 없어서, 바로 기존 시스템을 이어받는 상태는 아닙니다.
  3. Lovable은 새 전체 앱 후보 제작에는 쓸 수 있지만, 현재 검증된 서버 이미지를 그대로 호스팅하는 경로는 아닙니다.
  4. Render 후보는 render.yaml과 Dockerfile로 준비되어 있고, 서버는 호스팅 플랫폼의 PORT 값을 들을 수 있습니다.

5. 서버 이미지

  • 이미지 태그: ghcr.io/ethicsjayden31/bonsung-version3-server:codex-v1-intranet
  • 고정 digest: sha256:ca0f49d1e9df82b728807311f18a680fb14b7c3fb1f15cbb682267b14094f0d2
  • 외부 호스트에서는 /data 같은 영구 디스크를 붙이고 VERSION3_LOCAL_DATA_FILE=/data/version3-data.json을 사용합니다.
  • VERSION3_ALLOWED_ORIGINS는 https://ethicsjayden31.github.io로 제한해야 합니다.

6. 완료로 보지 않는 상태

  • 서버 주소가 http://127.0.0.1:4303 또는 localhost 같은 로컬 주소입니다.
  • 외부 HTTPS 서버 URL로 verify:version3-server를 통과하지 않았습니다.
  • 공개 UI가 검증된 VERSION3_API_BASE_URL을 사용하지 않습니다.
  • 운영 데이터 파일이 메모리나 컨테이너 임시 경로에 저장됩니다.
  • 실제 계정으로 로그인, 학생 계정 연결, 상담요청, 공지 작성, 데이터 내보내기/백업을 확인하지 않았습니다.

7. 다음 실행 순서

  1. 외부 HTTPS 서버를 띄우고 /health 응답을 확인합니다.
  2. GitHub Actions의 Verify External Version.3 Server에 서버 URL을 넣어 계약 검증을 통과시킵니다.
  3. Deploy GitHub Pages Preview를 같은 서버 URL과 함께 실행합니다.
  4. 배포가 통과하면 이 페이지에서 같은 서버 URL을 넣고 다시 확인합니다.
  5. 로그인 화면에서 실제 대표/매니저/강사/수강생 계정으로 운영 흐름을 확인합니다.
