본성뮤직 Version.3 구축 현황 점검

공개 UI와 별도 Version.3 서버가 실제 운영 흐름으로 연결될 준비가 되었는지 눈으로 확인하는 임시 페이지입니다. 최종 운영 안내 페이지가 아니며, 점검이 끝나면 제거할 수 있습니다.

1. Version.3 서버 연결 확인

서버 주소가 정해지면 ?api=https://서버주소를 붙여 열 수 있습니다. 예: version3-inspection.html?api=https://server.example.com

아직 서버 상태를 확인하지 않았습니다.

공개 UI

Next UI Version.3가 공개 경로를 대체하는 화면입니다.

서버 기준

별도 API Apps Script가 아닌 Version.3 서버가 로그인과 데이터를 담당해야 합니다.

점검 상태

대기 서버 health 응답을 확인하면 상태가 바뀝니다.

2. 화면 바로 열기

로그인 대시보드 계정 관리 학생 관리 상담요청 데이터 점검

3. 눈으로 확인할 것

  • 로그인 화면이 Version.3 서버 로그인을 우선 안내하는지 확인합니다.
  • 대표, 강사, 매니저, 수강생 역할 흐름이 분리되어 있는지 확인합니다.
  • 상담요청이 댓글식 Q&A가 아니라 일방향 요청과 상태 관리로 보이는지 확인합니다.
  • 공지 작성은 대표와 매니저 권한에서만 가능한지 확인합니다.
  • 데이터 점검 화면에서 내보내기, 가져오기, 감사 로그, 참조 무결성 흐름을 확인합니다.

5. 완료로 보지 않는 상태

  • 서버 주소가 http://127.0.0.1:4303 같은 로컬 주소이면 아직 실제 운영 배포가 아닙니다.
  • 서버 상태 확인이 실패하면 공개 UI가 있어도 별도 서버 운영 완료가 아닙니다.
  • Apps Script나 Preview 데이터가 기본 공개 경로에 노출되면 Version.3 공개 대체가 완료된 것이 아닙니다.
  • 실제 외부 HTTPS 서버 URL로 verify:version3-server를 통과해야 최종 운영 검증으로 볼 수 있습니다.

6. 공개 UI 연결 절차

  • 외부 HTTPS 서버가 준비되면 GitHub Actions에서 Deploy GitHub Pages Preview를 수동 실행합니다.
  • 이 작업은 서버 URL 형식과 실제 Version.3 API 동작을 먼저 검증합니다.
  • server_url에 입력한 주소가 검증을 통과하면 VERSION3_API_BASE_URL 배포 변수로 저장하고 GitHub Pages를 다시 배포합니다.
  • 재배포 후 이 점검 페이지에 같은 서버 주소를 넣어 /health 응답을 다시 확인합니다.
